### Effective Code Review Feedback

When giving code review feedback, focus on the following:

* **Specificity**: Clearly state the problem or issue you observed. Avoid general comments like "this is hard to read" – instead, say "the indentation is inconsistent in this block of code".
* **Relevance**: Ensure that your feedback is relevant to the code's current state. Avoid pointing out things that are not currently a problem or that will be addressed in a later PR.
* **Actionability**: Provide actionable suggestions for improvement. Instead of saying "improve this", say "consider using a more robust error handling mechanism".

Example of effective feedback:
```python
# instead of: "This is hard to read"
# Consider using a more consistent indentation style throughout the file
# instead of: "This is too long"
# Consider breaking up this long function into smaller, more focused ones
```
Effective feedback helps developers grow and improves overall code quality.
